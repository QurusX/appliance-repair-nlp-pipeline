# Задача: Автоматизация классификации и структурирования данных по ремонту бытовой техники

## 1. Контекст проекта
Заказчик обладает огромным массивом неструктурированных текстовых данных (сотни тысяч записей), включающим вопросы пользователей, ответы мастеров, статьи и посты с форумов по ремонту бытовой техники. Цель — создать инструмент для автоматической переработки этого «хаоса» в структурированную базу знаний.

## 2. Бизнес-цели
*   **Очистка данных:** Автоматически отсеивать тексты, не относящиеся к ремонту (вопросы о покупке, габаритах, выборе техники).
*   **Категоризация:** Определять тип неисправности для каждой записи (например, «Холодильник не морозит», «Стиральная машина не сливает воду»).
*   **Извлечение сущностей:** Выделять из текста важные технические данные:
    *   Модели устройств (Model numbers).
    *   Артикулы запчастей (Part numbers).
    *   Коды ошибок (Error/Fault codes).
*   **Оценка контента:** Помечать записи, которые содержат конкретные шаги решения проблемы или инструкции по ремонту.

## 3. Функциональные требования
*   **Локальная работа:** Скрипт должен работать полностью автономно без использования платных API (OpenAI, Anthropic и др.) на этапе массовой обработки.
*   **Масштабируемость:** Возможность пакетной (batch) обработки больших объемов данных.
*   **Гибкость:** Система должна легко адаптироваться к появлению новых моделей техники или типов поломок без жесткой перенастройки классификатора.
*   **Технологический стек:** Python, регулярные выражения (Regex), современные NLP-библиотеки (Transformers).

## 4. Исходные данные
1.  **Классификатор юнитов и проблем:** Базовый список типов техники и соответствующих им неисправностей.
2.  **Таблица Q&A (13k+ записей):** Массив пар «Вопрос-Ответ» для первичной обработки и обучения.
3.  **Примеры контента:** Образцы «эталонных» текстов с форумов и YouTube для настройки качества.

## 5. Ожидаемый результат (MVP)
Python-скрипт, который на входе принимает Excel/CSV файл, а на выходе выдает расширенную таблицу со следующими полями:
*   `Is_Repair` (bool) — относится ли к ремонту.
*   `Category` (string) — сформулированная проблема.
*   `Has_Solution` (bool) — наличие шагов решения.
*   `Model_Numbers` — список найденных моделей.
*   `Part_Numbers` — список найденных артикулов.
*   `Error_Codes` — найденные коды ошибок.
